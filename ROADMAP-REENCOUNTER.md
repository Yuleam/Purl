# 뜨개질 — 다시 만나기 위한 도구

> 이 도구의 가치는 잡는 순간이 아니라, 다시 만나는 순간에 발생한다.

## 설계 철학

| 원칙 | 의미 |
|------|------|
| **가볍게 잡는다** | 캡처는 씨앗을 심는 행위. 부담이 없어야 한다. |
| **다시 만나게 한다** | 과거의 조각이 현재의 필요와 자연스럽게 만나는 접점을 만든다. |
| **대신하지 않는다** | 도구는 연결을 판정하거나 정리하지 않는다. 조용히 놓아두고, 깨달음은 사용자에게서 나온다. |

**다정함의 구체적 의미**: 막혀있는 사람, 흩어진 사람, 길을 잃은 사람이 찾아온다. 이 사람에게 도구 사용법을 요구하지 않는다. 먼저 맞이한다.

---

## 핵심 개념 (4개)

| 개념 | 사용자 용어 | 설명 |
|------|-------------|------|
| **조각 (fiber)** | 조각 | 잡은 문장 + 그 순간의 반응 |
| **연결 (link)** | 연결 | 2개 이상의 조각이 닿아있다 + 왜 |
| **만남 (encounter)** | — (화면 이름 없음) | 과거 조각과의 재회 |
| **궤적 (trail)** | 궤적 | 내 조각들의 시간적 흐름 |

실(thread), 코(stitch), 편물(fabric)의 3단계 위계를 **연결 하나로 통합**한다. 사용자에게 연결의 종류를 구분하게 하지 않는다. "이것들이 닿아있다 + 왜" — 이게 전부.

---

## 조각의 구조

```
조각 {
  text        — 잡은 문장
  thought     — 그때 든 생각 (선택)
  감도        — 1~100, 얼마나 강하게 반응했는가
  방향성      — 공감 / 비판 / 보류
  출처        — 어디서 잡았는가 (선택)
  출처범위    — 원문 내 위치 (선택)
  created_at  — 잡은 시점
}
```

이 메타데이터가 단순한 부가정보가 아니다. 인지과학적으로 **인출의 열쇠**(Tulving, 1973)다. 나중에 다시 만날 때, 이 맥락이 함께 보여야 그때의 사고가 복원된다.

## 사용자 경험 — 세 가지 장면

### 장면 1: 만남 (앱을 열 때)

**의도**: 길을 잃은 사람, 그냥 들른 사람을 다정하게 맞이한다.

```
┌─────────────────────────────┐
│                             │
│   3주 전, 이런 조각을 잡았어  │
│                             │
│   "모든 분류 체계는 세계관의  │
│    반영이다"                 │
│                             │
│   감도 ■■■■■■■□□□  72       │
│   공감                      │
│   💭 분류 자체가 중립이 아닌  │
│      해석이라는 생각          │
│   📖 루만 카드상자 읽기, p.47 │
│                             │
│         [ 다음 만남 ]         │
│                             │
│   ─────────────────────     │
│   [ 잡기 ]    [ 궤적 ]       │
└─────────────────────────────┘
```

**설계 규칙**:
- 앱을 열면 **빈 화면 없이** 과거 조각 하나가 먼저 보인다
- 조각의 **전체 맥락**(감도, 방향성, 생각, 출처, 시점)을 함께 보여준다 — 부호화 맥락 복원
- "다음 만남"으로 다른 조각을 볼 수 있다. 의무 없음, 죄책감 없음
- 아래에 잡기와 궤적으로의 진입점

**선택 알고리즘** (Readwise 확률 모델 차용, LLM 없음):

```
각 조각에 가중치 부여:

  기본 가중치 = 1.0

  // 오래 안 본 조각일수록 높은 가중치
  if 한번도 만남 없음:
    가중치 × 3.0
  else:
    경과일 = (현재 - 마지막_만남) / 하루
    가중치 × (log(경과일 + 1) + 1)

  // 최근에 잡은 조각은 약간의 부스트 (신선함)
  if 잡은지 7일 이내:
    가중치 × 1.5

  // 감도가 높았던 조각은 약간의 부스트
  가중치 × (1 + 감도/200)

가중치에 비례하는 확률로 하나 선택
방향성 다양성 유지: 직전 만남과 다른 방향성 우선
```

### 장면 2: 잡기 + 주변부 (새 조각을 잡을 때)

**의도**: 잡는 행위는 가볍게. 그 순간 과거가 조용히 옆에 온다.

```
┌─────────────────────────────┐
│  ✕                    저장   │
│                             │
│  ┌───────────────────────┐  │
│  │ 텍스트 입력            │  │
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  감도  ━━━━━━━●━━━  72      │
│                             │
│  ◉ 공감  ○ 비판  ○ 보류     │
│                             │
│  💭 생각 (선택)              │
│  ┌───────────────────────┐  │
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  📖 출처 (선택)              │
│  ┌───────────────────────┐  │
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄ │
│                             │
│  과거의 조각                 │
│  ┌───────────────────────┐  │
│  │ "패턴은 반복에서 온다"  │  │
│  │  비판 · 감도 65 · 2주전 │  │
│  └───────────────────────┘  │
│                             │
└─────────────────────────────┘
```

**설계 규칙**:
- 캡처 영역과 주변부 영역은 **시각적으로 분리** (점선, 톤 차이)
- 주변부 조각은 1~2개. 절대 많지 않게
- **"비슷합니다", "유사도 87%", "관련 조각" 같은 라벨을 붙이지 않는다**. 그냥 있을 뿐이다. 판정은 사용자의 몫
- 주변부 조각을 탭하면 상세 보기 → 거기서 연결 가능

**주변부 선택 로직**:

```
사용자가 텍스트 입력 중 (200ms 디바운스):

  1순위: 현재 텍스트와 키워드 겹침 (단순 형태소 매칭)
  2순위: 같은 출처의 다른 조각
  3순위: 같은 방향성의 최근 조각

  → 1개 이상 매칭 시, 가장 관련도 높은 1~2개 표시
  → 매칭 없으면 주변부 비워둠 (억지로 채우지 않는다)
```

임베딩 유사도는 보조 수단으로 활용할 수 있지만, **제시 방식이 핵심**이다. "유사합니다"가 아니라 그냥 조용히 놓아둔다.

### 장면 3: 궤적 (내 조각들의 흐름)

**의도**: 흩어진 생각의 형태를 보여준다. 정리가 아니라 비춤.

```
┌────────────────────────────────────────────┐
│  궤적                          이번달  전체  │
│                                            │
│  3월 ─────────────────────────────────     │
│                                            │
│  11일  ●●           ● = 공감               │
│  10일  ◆            ◆ = 비판               │
│   9일  ●◆◆●         ◇ = 보류               │
│   8일  ◇                                   │
│   7일  ●●●◆         크기 = 감도             │
│   6일                                      │
│   5일  ◆◇●●                                │
│   4일  ●                                   │
│   3일  ◆◆◆●●◇◇                             │
│   2일  ●                                   │
│   1일  ●◇                                  │
│                                            │
│  2월 ─────────────────────────────────     │
│  28일  ◆●                                  │
│  ...                                       │
│                                            │
│  ──────────────────────────────────────    │
│  이번 달: 조각 23개                         │
│  공감 12 · 비판 7 · 보류 4                  │
│  평균 감도 58                               │
└────────────────────────────────────────────┘
```

**설계 규칙**:
- 시간 순서가 축이다. 위에서 아래로 흐른다
- 각 조각은 **점 하나**. 색 = 방향성, 크기 = 감도
- 점을 탭하면 해당 조각의 전체 맥락 표시
- 하단에 간단한 요약 (개수, 방향성 분포, 평균 감도) — 패턴을 읽는 보조
- **분류, 폴더, 태그 없음**. 시간과 반응만으로 형태가 드러난다

**데스크톱 확장 (explorer.html)**:

```
┌──────────────────┬──────────────────────────┐
│    궤적           │   선택한 조각              │
│                  │                          │
│  3/11 ●●         │  "모든 분류 체계는        │
│  3/10 ◆    ←선택  │   세계관의 반영이다"       │
│  3/9  ●◆◆●       │                          │
│  3/8  ◇          │  감도 72 · 공감            │
│  3/7  ●●●◆       │  💭 분류 자체가 중립이      │
│  ...             │     아닌 해석이라는 생각    │
│                  │  📖 루만 카드상자, p.47     │
│                  │                          │
│                  │  ── 연결된 조각 ──         │
│                  │  "패턴은 반복에서 온다"     │
│                  │   → "반복은 선택이다"       │
│                  │   왜: 분류도 반복된 선택의  │
│                  │       결과라는 생각        │
│                  │                          │
│                  │  [ 연결하기 ]              │
└──────────────────┴──────────────────────────┘
```

## 연결 — 단순한 하나의 행위

실(thread), 코(stitch), 편물(fabric) 3단계 위계 → **연결 하나로 통합**.

```
연결 {
  members[]   — 2개 이상의 조각
  why         — 왜 닿아있다고 느꼈는가 (사용자의 언어)
  created_at  — 연결을 만든 시점
}
```

**연결하는 방법** (모든 경로가 같은 행위):
1. 만남에서 과거 조각을 보다가 → "이거, 저것과 닿아있네" → 연결하기
2. 잡기에서 주변부 조각을 보다가 → 탭 → 연결하기
3. 궤적에서 두 점을 보다가 → 멀티 선택 → 연결하기
4. 조각 상세에서 → 연결하기 → 다른 조각 검색/선택

```
┌─────────────────────────────┐
│  연결하기                    │
│                             │
│  ┌───────────────────────┐  │
│  │ "모든 분류 체계는..."  │  │
│  │ "패턴은 반복에서..."   │  │
│  │          [ + 조각 추가 ]│  │
│  └───────────────────────┘  │
│                             │
│  왜 닿아있다고 느꼈나요?     │
│  ┌───────────────────────┐  │
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│          [ 연결 ]            │
└─────────────────────────────┘
```

## 데이터 모델

```sql
-- 조각 (기존 fibers 유지)
fibers (
  id, text, thought, tension, tone,
  source, source_range, created_at, updated_at
)

-- 조각 답글 (기존 유지)
fiber_replies (
  id, fiber_id, text, created_at
)

-- 연결 (threads + stitches + fabrics → 통합)
links (
  id, why, created_at
)

link_members (
  link_id, fiber_id, sort_order
)

-- 만남 기록 (신규)
encounters (
  id, fiber_id, encountered_at
)

-- 임베딩 (기존 유지, 주변부 보조용)
fiber_embeddings (
  fiber_id, embedding
)
```

마이그레이션: 기존 threads, stitches, fabrics의 모든 연결을 links로 변환. why 필드가 이미 있으므로 자연스럽게 매핑.

## API 설계

```
조각:
  POST   /api/fibers          — 새 조각 잡기
  GET    /api/fibers/:id      — 조각 상세
  GET    /api/fibers?search=  — 텍스트 검색
  PATCH  /api/fibers/:id      — 수정
  DELETE /api/fibers/:id      — 삭제

연결:
  POST   /api/links           — 연결 만들기 { members: [id, id, ...], why }
  GET    /api/links/:id       — 연결 상세
  GET    /api/fibers/:id/links — 이 조각의 모든 연결
  DELETE /api/links/:id       — 연결 삭제

만남:
  GET    /api/encounter       — 만남 조각 1개 (확률 선택)
  POST   /api/encounter       — 만남 기록 저장 { fiber_id }

주변부:
  GET    /api/periphery?text=&tone=&source=  — 주변부 조각 1~2개

궤적:
  GET    /api/trail?from=&to= — 기간 내 조각 목록 (시각화용)
  GET    /api/trail/summary?from=&to= — 기간 요약
```

## 화면 구조

### 모바일 (capture.html — PWA)

```
세 개의 탭:

  [ 만남 ]  [ 잡기 ]  [ 궤적 ]

만남 탭 (홈):
  → 과거 조각 1개 + 전체 맥락
  → 다음 만남 버튼
  → 조각 탭 시 상세 → 연결 보기/만들기

잡기 탭:
  → 캡처 폼 (텍스트, 감도, 방향성, 생각, 출처)
  → 하단 주변부 영역 (0~2개 과거 조각)
  → Share Target 진입 시 자동으로 이 탭 + 텍스트 채움

궤적 탭:
  → 시간순 점 시각화
  → 하단 요약
  → 점 탭 시 상세
```

### 데스크톱 (explorer.html)

```
2컬럼:

  좌: 궤적 (시간순 점 시각화, 스크롤)
  우: 상세 패널 (선택한 조각 + 연결 + 만남 기록)

  상단 바: [만남 모드] [궤적 모드] [잡기]

만남 모드:
  좌: 만남 히스토리 (이전에 만났던 조각들)
  우: 현재 만남 조각 상세
```

## 사용자 여정 — 세 가지 필요 상태

### 막혀있을 때

```
앱 열기 → 만남 (과거 조각 하나)
  → 직접 관련 있으면: "아, 이거!" → 연결
  → 관련 없으면: 잡기 탭으로 이동
    → 현재 고민을 텍스트로 적기 시작
    → 주변부에 관련 과거 조각 등장
    → "어, 이것..." → 깨달음
```

### 흩어져 있을 때

```
앱 열기 → 궤적 탭
  → 최근 2주간 점들의 분포를 봄
  → "비판 방향 조각이 이렇게 많았구나"
  → 특정 날짜에 몰린 점들을 탭
  → 조각들을 하나씩 읽으며 형태 발견
  → 필요하면 연결 만들기
```

### 길을 잃었을 때

```
앱 열기 → 만남 (과거 조각 하나)
  → "2달 전에 이런 걸 잡았구나"
  → 생각(thought)을 읽으며 그때의 나를 만남
  → 다음 만남 → 또 다른 과거의 나
  → 반복하다 보면 "나는 계속 이 방향을 향하고 있었구나"
  → 자기 연속성 회복
```

## 하지 않는 것들

| 하지 않는다 | 이유 |
|-------------|------|
| 유사도 점수 표시 | 판정은 사용자의 몫 |
| 자동 분류/태깅 | 도구가 대신하는 것 |
| 연결 종류 구분 (실/코/편물) | 분류의 부담, 깨달음의 촉진이 아님 |
| 빈 홈 화면 | 길 잃은 사람을 맞이하지 않는 것 |
| 그래프 뷰 (노드+엣지) | 탐색 방법을 강제함, 도구 중심 사고 |
| 복잡한 검색/필터 | 막혀있는 사람에게 도구 사용법을 요구함 |
| 리뷰 의무/스트릭 | 죄책감은 다정하지 않다 |
